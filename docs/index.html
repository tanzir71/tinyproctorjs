Integrity signals for browser-based exams.

Track tab switching, focus loss, fullscreen exits, and DevTools heuristics — plus a 30s heartbeat. Reports are sent via Beacon API or fetch(keepalive) to a collector endpoint for review.

Workflow

Embed → monitor → score → send. Built for low-friction integration and review.

Tab switching

Records visibility changes; increments a violation when the page becomes hidden.

Focus integrity

Tracks blur/focus; blur increments a violation to flag likely app switching.

Fullscreen exits

Detects fullscreen state changes; exiting fullscreen can be weighted heavily.

Heartbeat + score

Sends an “active” ping every 30 seconds with violation_count and integrity_score.

How it works

Local state is calculated client-side; the backend stores events for later review.

Local violation state

Maintains counts and timestamps per violation key (tab_hidden, window_blur, fullscreen_exit, devtools_*).

state: counts • timestamps • sessionId

Integrity score

Computes a score from configurable weights: 100 − Σ(weight × count), clamped to [0, 100].

score: 0..100 • weights configurable

Delivery model

Queues events locally and flushes in small batches; uses Beacon API or keepalive fetch on exit.

send: beacon → fetch(keepalive)

DevTools heuristics

Best-effort suspicion signals using key combos and window resize deltas. Not bypass-proof.

heuristics: keys • resize

Tech

Single JS file on the exam page; collector endpoint stores events for dashboard review.

Client

Embeddable script tag auto-init with data-* attributes (endpoint, attemptId, examId, weights).

Backend

Wide compatibility: accept JSON batches via POST; store into SQLite; render sessions and timelines.
