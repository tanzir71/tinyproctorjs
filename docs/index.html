Eleven signals, one integrity score

Every signal is individually configurable and weighted. The score (100 → 0) gives reviewers instant triage; the event timeline gives them evidence.

How it works

Local state is scored client-side; your backend stores the event stream for review. Delivery survives page close via the Beacon API.

1 · Embed

Drop the script tag on the exam page with your endpoint and attempt ID. Auto-initializes — no build step, no framework.

2 · Monitor

Passive listeners track visibility, focus, fullscreen, clipboard, keys and activity. No camera, mic, or keystroke contents — ever.

3 · Score

Configurable weights produce a live 0–100 integrity score: 100 − Σ(weight × count).

4 · Report

Events batch to your collector via sendBeacon / keepalive fetch with retry and backoff. Offline-tolerant.

What it deliberately doesn't do

Honest scope beats inflated promises. Client-side signals are indicators for human review — not proof, and not prevention.

No surveillance

No camera, no microphone, no screen recording, no keystroke logging, no clipboard contents. Events describe that something happened, never what was typed or copied.

No false security

A technically skilled candidate can disable any in-page script. tinyproctor.js deters casual misuse and produces an audit trail — for high-stakes certification, layer it with stronger controls.
