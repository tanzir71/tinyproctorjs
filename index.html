SYS · 01

The whole system, on one sheet

No vendor cloud in the loop. The script watches the exam page, scores what it sees, and ships the evidence to an endpoint you own.

01 / EMBED
One script tag

Auto-initializes from data-attributes. No build step, no framework, no install.

02 / MONITOR
Passive listeners

Visibility, focus, fullscreen, clipboard, keys, activity. Never camera, mic, or contents.

03 / SCORE
Weighted locally

Configurable weights produce a live 0–100 score for instant triage.

04 / REPORT
Batched to you

sendBeacon / keepalive fetch with retry + backoff. Survives page close. Offline-tolerant.

SYS · 02

Set up in the next five minutes

Two steps however you work: get the file, add the tag. Most people just hand step one and two to their AI agent.

Recommended

Ask your AI agent

Using Claude, Copilot, Cursor or any coding agent? Paste this prompt — it handles the file, the tag, and the collector endpoint for your stack.

Add tinyproctor.js exam monitoring to my project: 1. Fetch https://tanzir71.github.io/tinyproctorjs/tinyproctor.min.js into my static assets. 2. Add its script tag to my exam page, data-endpoint pointing at a new /collect route. 3. Build /collect per https://tanzir71.github.io/tinyproctorjs/llms.txt and store the events.

Works with any agent that can read URLs — the docs are agent-readable at /llms.txt.

One command

Grab the file

The library is a single static file — download it straight into your project, then add the script tag from the hero above.

# download the library into your assets curl -LO https://tanzir71.github.io/tinyproctorjs/tinyproctor.min.js # optional: full repo with demo server + tests git clone https://github.com/tanzir71/tinyproctorjs

Then add the script tag from the manual path — that's the whole client-side integration.

Manual

Copy the tag

Already have the file hosted? Drop the tag on the exam page and point it at your collector. That's the full client-side integration.

<script src="/js/tinyproctor.min.js" data-tinyproctor data-endpoint="https://your-site.com/collect.php" data-exam-id="FINAL_2026" data-attempt-id="ATTEMPT_1001"> </script>

Collector in 30 lines: copy-paste PHP or Node examples from the docs.

SYS · 03

Eleven signals, one integrity score

Every signal is individually configurable and weighted. The score gives reviewers instant triage; the event timeline gives them evidence.

Tab switching

Detects when the exam tab is hidden or the browser is minimized.

tab_hidden

Focus loss

Flags clicks into other apps and windows — the classic second-screen tell.

window_blur

Fullscreen exits

Enforce fullscreen at start, get an event the moment a candidate leaves it.

fullscreen_exit

Copy / cut / paste

Detects clipboard use anywhere on the page — and can block it outright.

copy_attempt · paste_attempt

Right-click

Context-menu detection with an optional hard block.

context_menu

Print attempts

Catches Ctrl/Cmd+P and the browser's print dialog.

print_attempt

DevTools heuristics

Key-combo and window-resize heuristics flag console snooping. Best-effort by design.

devtools_key · devtools_resize

Idle + heartbeat

Inactivity detection plus a 30s heartbeat so dropped sessions stand out server-side.

page_idle · heartbeat

SYS · 05

What it deliberately doesn't do

Honest scope beats inflated promises. Client-side signals are indicators for human review — not proof, and not prevention.

No surveillance

No camera, no microphone, no screen recording, no keystroke logging, no clipboard contents. Events describe that something happened, never what was typed or copied.

No false security

A technically skilled candidate can disable any in-page script. tinyproctor.js deters casual misuse and produces an audit trail — for high-stakes certification, layer it with stronger controls.
